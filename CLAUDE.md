# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this repo is

Infrastructure-as-config for the Stoganet home server. **No application code, no build system, no test suite** — just shell scripts, a Docker Compose file, a systemd unit, and JSON config snippets applied to running containers. Sister repo `Stoganet/stogad` holds the native daemon (Rust).

Two top-level deliverables:

- `compose/` — Docker Compose stack (Traefik, Jellyfin, *arr stack, Gluetun+qBittorrent, Jellyseerr, Portainer) plus `setup.sh` (host bootstrap) and `configure-arr.sh` (idempotent post-deploy config of Sonarr/Radarr/qBittorrent via their REST APIs using JSON in `compose/arrconfig/`).
- `backup/` — LUKS-encrypted USB backup driven by a systemd timer. `setup.sh` is one-time drive init (LUKS2 + ext4 + keyfile); `backup.sh` runs daily, opens LUKS, rclones Docker volumes (read from `/var/lib/docker/volumes/${COMPOSE_PROJECT}_*_config/_data`), and sends ntfy alerts.

## Architecture invariants

These cross-file constraints matter when editing:

- **Compose project name is `services`** (set by `name: services` in `compose/docker-compose.yml`). `backup/backup.sh` defaults `COMPOSE_PROJECT=services` and reads volumes at `${DOCKER_VOLUME_PATH}/${COMPOSE_PROJECT}_<volume>_config/_data`. Renaming the project breaks backups silently.
- **No WAN exposure.** Traefik binds only to `127.0.0.1` and `${NETBIRD_IP}` (the wt0 mesh interface). No port forwarding. All clients reach services via NetBird. Don't add `0.0.0.0` bindings.
- **TLS = wildcard via DNS-01** (`*.${DOMAIN}`). Subdomain names never appear in Certificate Transparency logs. Use the wildcard router pattern from the Traefik service block when adding a new subdomain — don't request per-host certs.
- **qBittorrent has no network of its own.** It uses `network_mode: service:gluetun`. Its WebUI port and Traefik labels live on the `gluetun` service, not `qbittorrent`. If Gluetun's tunnel drops, all torrent egress dies (kill switch).
- **Media path is `/mnt/wd/media`** on the host, mounted into Jellyfin (`:ro`), qBittorrent, Sonarr, Radarr, Bazarr (all `:rw` at `/data`, except Jellyfin at `/media`). Keep this path consistent across services so the *arr stack's hardlinks work.
- **Secrets in `compose/.env`** (gitignored), generated by `compose/setup.sh`. Never commit `.env`. The setup script preserves existing values on re-run via `.env.bak`.

## Common operations

```bash
# Bring stack up / down (run from compose/)
docker compose up -d
docker compose logs -f <service>
docker compose pull && docker compose up -d   # update images

# Re-apply Sonarr/Radarr/qBittorrent settings (idempotent, safe to re-run)
./configure-arr.sh

# One-time host bootstrap (Docker, NetBird, UFW, ZRAM, GPU drivers, .env, traefik.yml)
sudo ./setup.sh

# Backup ops (on host)
sudo systemctl start backup.service        # run backup now
sudo journalctl -u backup.service -f       # tail backup log
sudo systemctl list-timers backup.timer    # next scheduled run
```

There is no lint/test command — shell scripts use `set -euo pipefail`. If you need to validate, run `shellcheck` and `docker compose config` manually.

## Editing notes

- `compose/setup.sh` is **destructive on first run** (UFW reset, SSH config edit, swap removal). It guards with `ufw status | grep -qw active` and an `if [ -f /etc/ssh/sshd_config ]` check, but treat changes carefully. Re-runs preserve `.env` and `traefik.yml`.
- `configure-arr.sh` auto-detects API keys by `docker exec`-ing into containers and grepping `/config/config.xml`. If you add a new arr service, follow the same pattern rather than requiring keys in `.env`.
- `backup/backup.sh` has a `cleanup` trap that unmounts and closes LUKS on any exit. Don't add `exit` paths that bypass it; let the trap fire.
- The `.github/workflows/homed.yml` builds a Rust binary from `home/homed/` — that directory was extracted to the `stogad` repo (see commit `c96d879`). The workflow is currently stale; don't extend it without confirming intent.

## Stoganet conventions

Skills in `.claude/skills/` enforce the org-wide workflow — invoke them via the Skill tool, don't reimplement:

- **committing** — Conventional Commits; `stogad` uses release-please so prefixes drive versioning. This repo doesn't release, but match the style.

- **creating-issues** — Shared label vocabulary and Summary/Context/Scope/Acceptance/Notes body across all Stoganet repos.
